name_alias_for_entities_prompt: Used specifically for extracting and generating alias variations for company or organization names.
primary_name_entities: A prompt used to identify the main/primary name of an entity when multiple variations or aliases are present in the text.
2. Name Cleaning and Splitting (Pure/Impure Names)
In utils/eu_utils/eu_name_aliases_utils.py, the code makes sequential LLM calls to clean up messy name strings:

pure_impure_prompt: The LLM evaluates if a name is "pure" (just the name) or "impure" (contains titles, ranks, or other non-name text) using llm_call_and_parse_json_dict_response.
Part Splitting: For complex strings, the LLM splits the text into constituent parts (e.g., separating the actual name from titles like "General" or "Director") and appends the results to first, middle, last, and whole name arrays.

In utils/eu_pdf_utils/eu_pdf_data_points_extractor_utils.py, when extracting structured data points from the raw PDF text, the LLM is given context about the "Last Entity" processed.

The code passes *Last Entity*: {last_entity.get("name", "")} into the LLM prompt. This ensures that the LLM understands which person or company the current chunk of text refers to, preventing data from being attributed to the wrong name.

Based on the codebase and our previous findings, the phrase "name engine" doesn't literally appear as a class or folder name in the code. However, conceptually, the "name engine" refers to the highly orchestrated pipeline of LLM calls housed primarily within utils/eu_utils/eu_name_aliases_utils.py and powered by utils/eu_utils/eu_prompts.py.

This system acts as an "engine" because it takes raw, messy name strings from press releases and puts them through a multi-step assembly line of LLM processing to output clean, structured data.

Here is what the "Name Engine" consists of in this codebase:

The Classifier (pure_impure_prompt): It first acts as a filter, taking a raw string and determining if it's a "pure" name (just first/last name) or an "impure" name (cluttered with titles, ranks, or locations).

The Part Splitter (part_split_call): For impure names or complex strings, it acts like a parsing engine, breaking the string down into precise constituent parts (e.g., separating "General" from "John Doe" or separating Russian patronymics).

The Alias Generator (name_alias_for_individuals_prompt & name_alias_for_entities_prompt): This is the core of the engine. It takes the cleaned name parts and algorithmically generates a comprehensive dictionary of all possible name aliases (first, middle, last, and whole name combinations).

The Primary Extractor (primary_name_entities): When multiple names or organizations are tangled together, this part of the engine identifies which is the primary subject.

In short, eu_name_aliases_utils.py is the "Name Engine". It acts as the central brain that orchestrates these sequential LangChain/OpenAI calls to transform unstructured, messy text into highly structured, alias-rich entity profiles.

from pydantic import BaseModel, Field
from langchain_core.output_parsers import JsonOutputParser

class EdgeCaseNameResult(BaseModel):
    is_edge_case: bool = Field(description="True if the name contains extreme edge cases like multiple languages mixed, mononyms, or severe formatting issues.")
    cleaned_names: list[str] = Field(description="List of cleaned and standardized names extracted from the edge case input.")

def edge_case_name_call():
    system_prompt = """
    You are an expert at analyzing and resolving complex and edge-case name structures from press releases.
    Your task is to identify if the given name string contains extreme edge cases (e.g., dual language combinations like 'John Doe (Иван Доев)', 
    mononyms, excessive or malformed punctuation, or titles embedded directly into the name without separation) and clean it.
    
    If it is an edge case, extract all valid, distinct names and return them in a standardized, clean format in the `cleaned_names` array.
    If it is NOT an edge case, return the original name in the `cleaned_names` array.
    """
    parser = JsonOutputParser(pydantic_object=EdgeCaseNameResult)
    format_instructions = parser.get_format_instructions()
    return system_prompt, format_instructions, parser

import itertools
import difflib
import concurrent.futures
from utils.eu_utils.eu_common_utils import get_logger, llm_for_pydantic
from utils.llm_utils.lang_chain_utils import execute_lang_chain
from utils.eu_utils.eu_prompts_v2 import edge_case_name_call

logger = get_logger("eu_name_engine_v2")

def detect_and_clean_edge_cases(raw_names):
    """
    Passes names through an LLM to identify and resolve complex edge cases.
    """
    cleaned = []
    system_prompt, format_instructions, parser = edge_case_name_call()
    
    for name in raw_names:
        try:
            response, _ = execute_lang_chain(
                system_prompt=system_prompt,
                format_instructions=format_instructions,
                input_content=f"INPUT NAME: {name}",
                parser=parser,
                model=llm_for_pydantic
            )
            # Response is a dictionary due to JsonOutputParser mapping to Pydantic
            if response.get("is_edge_case") and response.get("cleaned_names"):
                cleaned.extend(response.get("cleaned_names"))
            else:
                cleaned.append(name)
        except Exception as e:
            logger.error(f"Error in edge case detection for {name}: {e}")
            cleaned.append(name)
            
    return list(set(cleaned))

def generate_permutations(cleaned_names):
    """
    Generates permutations and combinations of the name parts.
    """
    permutations_set = set()
    for name in cleaned_names:
        parts = name.split()
        if len(parts) <= 1:
            permutations_set.add(name)
            continue
            
        # Generate permutations of all lengths up to the number of parts
        for r in range(1, len(parts) + 1):
            for perm in itertools.permutations(parts, r):
                permutations_set.add(" ".join(perm))
                
    return list(permutations_set)

def search_and_rank_names(permutations, reference_doc):
    """
    Searches the generated permutations against a reference doc (list of strings)
    and returns the top 10 most similar matches using SequenceMatcher.
    """
    scored_matches = []
    
    for perm in permutations:
        for ref_name in reference_doc:
            # Calculate similarity ratio between 0 and 1
            ratio = difflib.SequenceMatcher(None, perm.lower(), ref_name.lower()).ratio()
            if ratio > 0.6:  # Threshold to filter out complete mismatches
                scored_matches.append({
                    "input_permutation": perm,
                    "matched_reference": ref_name,
                    "score": ratio
                })
                
    # Sort by score descending
    scored_matches.sort(key=lambda x: x["score"], reverse=True)
    
    # Deduplicate keeping highest scores
    seen_refs = set()
    unique_top_matches = []
    for match in scored_matches:
        if match["matched_reference"] not in seen_refs:
            seen_refs.add(match["matched_reference"])
            unique_top_matches.append(match)
            if len(unique_top_matches) >= 10:
                break
                
    return unique_top_matches

def _run_pipeline_sync(raw_names, reference_doc):
    logger.info(f"Starting Name Engine v2 for names: {raw_names}")
    
    # Step 1: LLM Edge Case Detection
    cleaned_names = detect_and_clean_edge_cases(raw_names)
    logger.info(f"Cleaned names from edge cases: {cleaned_names}")
    
    # Step 2: Generate Permutations
    permutations = generate_permutations(cleaned_names)
    logger.info(f"Generated {len(permutations)} permutations.")
    
    # Step 3: Search and Rank
    top_matches = search_and_rank_names(permutations, reference_doc)
    logger.info(f"Found {len(top_matches)} top matches.")
    
    return {
        "original_names": raw_names,
        "cleaned_names": cleaned_names,
        "permutation_count": len(permutations),
        "top_10_similar_matches": top_matches
    }

def run_name_engine_v2_pipeline(raw_names, reference_doc):
    """
    Entry point to run Name Engine v2 in parallel to the main thread.
    Returns a concurrent.futures.Future object.
    """
    executor = concurrent.futures.ThreadPoolExecutor(max_workers=1)
    future = executor.submit(_run_pipeline_sync, raw_names, reference_doc)
    return future

        lines = data.strip().split('\n')
        found_brace = False
        for i in range(len(lines) - 1, -1, -1):
            if lines[i].strip() == '},' and 'id_remarks' not in lines[i - 1] and "wholeNameVariations" not in lines[i-1]:
                found_brace = True
                break

        if found_brace:
            output_lines = lines[:i+1]
            output_string = '\n'.join(output_lines)
            output_string_ready_for_json_loads = output_string[:-1] + "]}"
            return output_string_ready_for_json_loads
        else:
            raise ExtractorException(f"No ',' found in the data. {data}")
    except Exception as E:
        raise ExtractorException(f"Error in getting json parsable string from llm output: {E}input is {data}")

How it runs in parallel
By calling run_name_engine_v2_pipeline(raw_names, []), it triggers a ThreadPoolExecutor inside eu_name_engine_v2_utils.py. This executor takes the task and runs it on a separate background thread.

This means that the original extraction process immediately continues working on the next task without waiting for the V2 LLM calls or ranking algorithms to finish.
